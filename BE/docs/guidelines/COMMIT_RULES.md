# 커밋 규칙

## 커밋 메시지 형식
`유형(<scope>): <subject>`

## 커밋 유형
- ✨ feat: 새로운 기능 추가
- 🐛 fix: 기능 수정, 버그 수정
- 💡 chore: 오타 수정, 코드 변경 (주석 추가/수정 포함)
- 📝 docs: 문서 수정
- 🚚 build: 빌드 관련 파일 수정/삭제
- ✅ test: 테스트 코드 추가
- ♻️ refactor: 코드 리팩터링

## 커밋 메시지 예시
- ✨ feat(auth): implement JWT authentication
- 🐛 fix(database): resolve connection timeout issue
- ♻️ refactor(api): optimize user query performance
